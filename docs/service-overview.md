# 지금빌려 — 서비스 소개

> **"그 책, 지금 어디서 빌릴 수 있지?"**
> 읽고 싶은 책이 생긴 순간, 나랑 가까운 도서관에서 지금 바로 빌릴 수 있는지 확인해 주는 모바일 웹 서비스.

---

## 1. 한 줄 소개

**지금빌려**는 책 제목 하나만 검색하면 ① 전자책으로 **지금 바로 읽을 수 있는지**, ② 종이책이라면 **내 주변 어느 도서관에 몇 권 남아 있는지**를 지도 위에서 한눈에 보여주는 도서관 대출 확인 서비스입니다.

---

## 2. 어떤 문제를 해결하나요?

책을 빌려 읽고 싶은 사람이 실제로 겪는 여정은 이렇습니다.

- 도서관마다 홈페이지가 따로 있어서 **한 권을 찾으려면 여러 사이트를 돌아다녀야** 합니다.
- 소장 여부는 나와도 **"지금" 대출 가능한지**는 확인이 번거롭습니다.
- 어렵게 확인하고 찾아갔는데 **방금 마지막 한 권이 나가버리는 헛걸음**도 생깁니다.
- 전자책은 도서관별 전자도서관이 또 따로 있어서 존재조차 모르는 경우가 많습니다.

지금빌려는 이 과정을 **검색 한 번**으로 줄입니다. 여러 도서관의 실시간 대출 정보를 한 화면에 모아서, "빌리러 갈지 말지"를 그 자리에서 결정할 수 있게 해 줍니다.

---

## 3. 핵심 사용자 여정

### 여정 요약

```
책 제목 검색
   ↓
① 전자책 검색 (기본 진입) — "지금 바로 읽을 수 있나?"
   ↓ 없거나 모두 대출중이면
② 종이책 검색 — "내 주변 도서관에 있나?" (GPS 기반)
   ↓ 책 선택
③ 지도 화면 — 도서관별 위치·대출가능 권수 확인
   ↓ 도서관 선택
④ 길찾기(카카오맵) 또는 도서관 홈페이지에서 최종 확인
```

### 단계별 상세

**① 전자책 먼저 (기본 화면)**

- 서비스에 들어오면 **전자책 검색**이 기본입니다. 전자책이 있으면 이동 없이 지금 바로 읽을 수 있기 때문입니다.
- 서울 시내 전자도서관 8곳(강남·구로·금천·동대문·마포·서초 구립전자도서관, 서울시 전자도서관, 서울시육아종합지원센터)을 **동시에 검색**합니다.
- 결과는 도서관별로 **"N권 대출가능" / "모두 대출중"** 으로 명확하게 표시하고, 탭 한 번으로 해당 전자도서관 페이지로 이동합니다.
- 검색 전 화면에는 **"요즘 뜨는 책" 추천 칩**이 떠서, 서울 공공도서관 20·30대 대출 데이터(정보나루) 기반 인기 도서를 바로 검색해 볼 수 있습니다.

**② 종이책으로 자연스럽게 전환**

- 전자책이 없으면 화면의 **"종이책으로 찾아보시겠어요?"** 버튼 하나로, 입력했던 검색어를 그대로 들고 종이책 검색으로 넘어갑니다. (반대 방향 전환도 동일하게 지원)
- 종이책 검색은 **GPS 위치 기반**입니다. 내 위치 주변(반경 약 5km)의 구(區)를 자동으로 골라 그 지역 도서관부터 검색하고, 위치 정보가 없으면 **서울 전체 25개 구**를 검색합니다.
- 검색 중에도 "지금 동작구, 서초구에서 검색 중..."처럼 **어디를 찾고 있는지 실시간으로** 보여줍니다.
- 결과 목록은 **소장 도서관 수·대출가능 도서관 수** 순으로 정렬되어, 빌릴 확률이 높은 판본이 위로 올라옵니다.

**③ 지도에서 한눈에**

- 책을 선택하면 **카카오맵 기반 지도**에 소장 도서관들이 마커로 표시됩니다.
- 마커에는 도서관 이름과 함께 **"2권" / "가능" / "대출중"** 이 바로 보여서, 지도만 봐도 어디로 가야 할지 판단할 수 있습니다.
- 지도를 다른 동네로 옮기면 **"이 지역에서 재검색"** 으로 이사·외출 중에도 그 지역 기준으로 다시 찾을 수 있습니다.

**④ 헛걸음 방지 설계로 마무리**

- 도서관 상세 카드는 **대출가능 권수에 따라 안내 버튼이 달라집니다.**
  - 2권 이상 여유: **"길찾기"** 가 주 버튼 → 카카오맵/카카오내비로 바로 연결
  - 1권뿐이거나 권수 미상: **"확인하기"** 가 주 버튼 → 마지막 한 권이 방금 나갔을 수 있으니 도서관 홈페이지에서 한 번 더 확인하도록 유도
  - 모두 대출중: **"확인하기"** → 예약 등 다음 행동으로 연결
- 상세 카드에는 주소, 거리, 운영시간, 전화번호와 함께 **그 책의 검색 결과 페이지로 바로 가는 링크**가 붙습니다.

## 4. 주요 기능 정리

| 기능 | 설명 |
|---|---|
| 전자책 통합 검색 | 서울 전자도서관 8곳 동시 검색, 도서관별 대출가능 권수 표시 |
| 종이책 위치 기반 검색 | GPS로 주변 구 자동 선택, 위치 없으면 서울 전체 검색 |
| 실시간 대출가능 여부 | 도서관 소장 여부가 아니라 "지금 빌릴 수 있는가"를 표시 |
| 지도 뷰 | 카카오맵 위에 도서관별 대출가능 권수 마커 표시 |
| 이 지역에서 재검색 | 지도를 옮기면 그 지역 기준으로 다시 검색 |
| 길찾기 연동 | 카카오맵/카카오내비 도보 길찾기 원터치 연결 |
| 전자책 ↔ 종이책 전환 | 검색어를 유지한 채 버튼 하나로 상호 전환 |
| 요즘 뜨는 책 추천 | 정보나루 20·30대 대출 데이터 기반 인기 도서 칩 |
| 헛걸음 방지 버튼 위계 | 대출가능 권수(여유/빠듯/불가)에 따라 길찾기·확인하기 우선순위 자동 조정 |
| 딥링크 지원 | `/ebook?q=책제목` 링크 공유 시 자동 검색 — 외부 채널(챗봇·SNS) 연동에 유리 |

## 5. 서비스 커버리지 & 데이터

- **지역**: 서울특별시 (종이책은 서울 전 지역, 위치 기반으로 주변 구 우선 검색)
- **전자책**: 서울 시내 전자도서관 8곳 통합 검색
- **데이터 소스**:
  - 각 도서관 통합검색 시스템 실시간 조회(스크래핑) — 실제 대출가능 **권수**까지 확보
  - **정보나루(도서관 정보나루, data4library.kr)** 공공 데이터 — 스크래핑이 어려운 구 보강 및 추천 도서 추출
  - 카카오 책 검색 / 카카오맵 API
- **고지 사항**: 실시간성이 완벽하지 않을 수 있어, 화면에 "실제 대출가능 여부는 도서관 홈페이지에서 다시 확인해 주세요"를 항상 안내합니다.

## 6. 제품이 지키는 UX 원칙 (톤앤매너 참고용)

1. **"지금"이 핵심** — 소장 목록이 아니라 "지금 빌릴 수 있는가"만 답한다. 전체 소장 권수처럼 의사결정에 불필요한 정보는 과감히 뺀다.
2. **가장 빠른 경로 먼저** — 이동이 필요 없는 전자책을 기본 진입으로, 없으면 종이책으로 자연스럽게 흘려보낸다.
3. **헛걸음을 막는다** — 마지막 1권일 때는 길찾기보다 재확인을 먼저 권한다. 사용자의 발걸음을 책임진다는 태도.
4. **친근한 말투** — "그 책, 제목이 뭐였더라?", "종이책으로 찾아보시겠어요?" 같은 구어체로 도서관 서비스 특유의 딱딱함을 걷어낸다.
5. **모바일 퍼스트** — 걸어가면서 쓰는 서비스. 지도 UX, 원터치 길찾기, 하단 고정 버튼 등 한 손 조작 기준으로 설계.

## 7. 타깃 사용자

- **서울 거주 20~40대 독서 인구** — 특히 신간·화제작을 사기 전에 빌려 보려는 사람
- **전자책 도서관의 존재를 모르는 사람** — "도서관 회원이면 전자책을 공짜로 볼 수 있다"는 발견의 순간을 제공
- **아이와 함께 도서관을 이용하는 양육자** — 어린이도서관·육아종합지원센터 자료 포함
- **자투리 시간 독서족** — 퇴근길에 "지금 들러서 빌릴 수 있는지"를 30초 안에 판단하고 싶은 사람

## 8. 마케팅 메시지 소재

- 메인 카피(현행): **"그 책, 지금 어디서 빌릴 수 있지?"**
- 서브 카피(현행): "나랑 가까운 도서관에서 지금 빌릴 수 있는지 바로 확인해 드려요."
- 소구 포인트:
  - "검색 한 번, 서울 도서관 전부" — 여러 도서관 사이트를 도는 수고의 제거
  - "사기 전에, 지금빌려" — 도서 구매 대비 절약 프레임
  - "전자책이면 이동도 필요 없어요" — 즉시성
  - "헛걸음은 저희가 막아드려요" — 마지막 1권 재확인 유도 설계
  - "요즘 뜨는 책도 바로" — 또래(20·30대) 대출 데이터 기반 추천
